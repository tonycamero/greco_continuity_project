# Greco Public Signal System

This folder is the parallel publishing system for the Greco Continuity Project.

It turns the repo's core assets into a daily public signal designed to prepare the market, name the category, and activate the relationship graph around each post.

## Operating Thesis

The public category is:

> Regenerative Coordination Economy

The Greco mechanism inside that category is:

> Trust-Mediated Exchange

The bridge line is:

> Community without exchange is fragile.

## Source Hierarchy

Use the source hierarchy in `../greco_core_source_alignment.md`:

1. `../The Greco Continuity Project.docx` - mission, stewardship, continuity.
2. `../THE END OF MONEY 2026 Revised Edition-Publisher Brief.docx` - publisher-facing case.
3. `../Chapter 21 Expansion FINAL.docx` - current travel asset.
4. Market and relationship files - support evidence, channels, and outreach.

## Files

- `posting_strategy.md` - category strategy, cadence, tone, and safety rules.
- `daily_publish_queue.md` - first 14 days of ready-to-use posts.
- `contact_notification_matrix.csv` - who to notify for each post.
- `templates/post_record_template.md` - reusable post planning template.

## Publishing Rule

Post publicly once per day on either LinkedIn or X.

For X, publish only as threads of 6-10 posts.

For LinkedIn, publish as one complete post with a clear category-building thesis.

After each public post, notify only the most relevant people or organizations. Keep notifications human-approved and relationship-first.
